## üìå Project Overview

The **Redbus Data Scraping and Filtering Application** is a comprehensive solution designed to automate the collection, storage, and visualization of bus travel data across various state transport and private operators from the [Redbus](https://www.redbus.in/) website.

The project uses **Selenium** for dynamic web scraping, stores the data in a **SQL database**, and provides an **interactive Streamlit application** to filter and explore the data. This enables real-time insights into bus schedules, prices, availability, and more ‚Äî empowering both end-users and stakeholders in the transportation domain.

---

## üéØ Problem Statement

The project aims to enhance the accessibility and usability of bus travel data by:
- Automating the extraction of Redbus route and schedule data.
- Providing interactive data filtering using Streamlit.
- Enabling better decision-making for travelers, aggregators, and analysts.

By streamlining these tasks, it significantly improves operational efficiency and supports strategic planning in the transport industry.

## üóÉÔ∏è Dataset Description

The dataset is generated by scraping the [Redbus](https://www.redbus.in/) website and includes:

| Field               | Description                                                |
|--------------------|------------------------------------------------------------|
| `route_name`        | Name of the bus route (e.g., "Hyderabad ‚Üí Vijayawada")     |
| `route_link`        | Direct link to the Redbus route                            |
| `busname`           | Bus operator or company name                               |
| `bustype`           | Bus type (e.g., Sleeper, AC, Seater)                       |
| `departing_time`    | Time the bus departs                                       |
| `duration`          | Total travel duration                                      |
| `reaching_time`     | Arrival time at the destination                            |
| `star_rating`       | Passenger rating (out of 5)                                |
| `price`             | Ticket cost (in INR)                                       |
| `seats_available`   | Number of seats available at the time of scraping          |

## üóÑ Model Database Schema

**Table Name**: `buses`

| Column Name       | Data Type       | Description                             |
|-------------------|------------------|-----------------------------------------|
| `id`              | INT (Primary Key, Auto Increment) | Unique identifier   |

| `busname`         | TEXT             | Bus operator name                       |
| `bustype`         | TEXT             | Type of bus                             |
| `departing_time`  | TIME             | Departure time                          |
| `duration`        | TEXT             | Travel duration                         |
| `reaching_time`   | TIME             | Arrival time                            |
| `star_rating`     | FLOAT            | Star rating from passengers             |
| `price`           | DECIMAL          | Ticket price                            |
| `seats_available` | INT              | Number of available seats               |

## Implemented Schema in MySQL

```sql
CREATE TABLE buses (
    id INT AUTO_INCREMENT PRIMARY KEY,
    state_name VARCHAR(100),
    route_name VARCHAR(200),
    busname VARCHAR(200),
    price DECIMAL(10, 2),
    bustype VARCHAR(100),
    departing_time VARCHAR(20),
    duration VARCHAR(20),
    reaching_time VARCHAR(20),
    star_rating VARCHAR(10),
    seats_available VARCHAR(50),
    type VARCHAR(50),
    date_of_scrape DATE
);
```

## üß≠ Approach

### 1. Web Scraping
- Use **Selenium** to automate browser actions.
- Navigate through state transport corporation routes.
- Scroll dynamically to handle **lazy loading**.
- Click sub-tabs for **government buses** (e.g., KSRTC, APSRTC).
- Extract bus-level details from each route.

### 2. SQL Data Storage
- Create schema with correct data types.
- Use `sqlite3` or `mysql-connector-python` to store the data.
- Ensure deduplication and data quality.

### 3. Streamlit App
- Connect the app to the database.
- Use **Streamlit widgets** to filter by:
  - Route Name
  - Bus Type
  - Price Range
  - Star Rating
  - Availability
- Display the filtered data in a clean table.
